# 은행 고객경험관리 시스템 매뉴얼

## 1. 고객경험관리(CXM) 체계 개요

### 1.1 고객경험관리의 목적

은행 고객경험관리(CXM)는 고객이 은행을 이용하는 전 과정에서 느끼는 **경험의 품질을 수치화·분석·개선**하는 체계이다.
이를 통해 다음을 달성한다.

* 타 은행 대비 경쟁력 진단
* 고객 불만의 구조적 원인 파악
* 고객 유형별 핵심 영향요인 도출
* 조직·지역·직원 단위 성과 관리
* 데이터 기반 합리적 개선과제 도출

---

## 2. 설문조사 체계 (Top Down & Bottom Up)

고객경험 조사는 **Top Down(TD)** 과 **Bottom Up(BU)** 방식으로 구분된다.

| 구분    | Top Down(TD)     | Bottom Up(BU)    |
| ----- | ---------------- | ---------------- |
| 조사 주기 | 반기 1회            | 매일 (상시)          |
| 조사 목적 | 타 은행과의 고객 만족도 비교 | 실제 이용 고객의 실경험 측정 |
| 조사 대상 | 여러 은행을 이용하는 일반 금융 소비자 | KB국민은행 채널을 실제 이용한 고객 |
| 조사 방식 | 외부 조사기관 위탁 (주거래은행별 응답) | 채널 이용 직후 자체 발송 |
| 조사 성격 | 인식 기반 조사 (경쟁사 비교 가능) | 실제 경험 기반 조사 (내부 개선용) |

### 2.1 TD와 BU의 핵심 차이

**Top Down(TD)**:
* 외부 조사기관을 통해 **여러 은행 고객을 대상**으로 설문
* 응답자가 주거래은행을 선택하므로 **은행 간 NPS 비교** 가능
* 채널별로 **벤치마크 은행(직상위 기관)** 산정 가능
* KB국민은행 기준 경쟁력 진단에 활용

**Bottom Up(BU)**:
* KB국민은행 채널 이용 직후 **자체 발송**하는 설문
* **실제 이용 고객**의 생생한 경험 수집
* 내부 개선과제 도출 및 조직/직원 성과 관리에 활용
* 타행 비교 불가 (자사 고객만 대상)

## 3. 고객경험 CASE조사

### 3.1 케이스 조사 개요

고객경험 케이스 조사는 **특정 주제나 이슈에 대해 심층적으로 파악**하기 위해 별도의 설문을 설계하여 진행하는 조사 방식이다.

| 구분 | 내용 |
|------|------|
| 조사 목적 | 특정 주제에 대한 고객 인식 및 경험 심층 파악 |
| 조사 방식 | 주제별 맞춤 설문 문항 설계 및 배포 |
| 조사 주기 | 필요 시 수시 진행 |
| 결과 활용 | 해당 주제에 대한 의사결정 및 개선방향 도출 |

### 3.2 케이스 조사 활용 예시

| 조사 주제 | 설문 내용 예시 | 기대 결과 |
|----------|---------------|----------|
| 신규 서비스 출시 전 조사 | 서비스 필요성, 예상 이용 의향, 기대 기능 | 출시 전 고객 수요 검증 |
| 특정 불만 이슈 심층 조사 | 불만 경험 상세, 개선 희망 사항, 대안 제시 | 이슈 원인 파악 및 해결책 도출 |
| 채널 개편 후 반응 조사 | 개편 인지도, 사용 편의성, 개선 요청 | 개편 효과 측정 및 추가 개선점 발굴 |
| 경쟁사 대비 인식 조사 | 타행 이용 경험, 비교 우위/열위 항목 | 경쟁력 강화 포인트 도출 |

### 3.3 케이스 조사 프로세스
```
1. 조사 주제 선정 → 2. 설문 문항 설계 → 3. 대상 고객 선정 → 4. 설문 배포 → 5. 결과 수집 → 6. 분석 및 보고
```

## 4. 영업점 자가진단

### 4.1 자가진단 개요

영업점 자가진단은 **조직 단위별로 NPS를 측정**하여 고객경험 성과를 자체적으로 진단하고 관리하는 체계이다.

| 구분 | 내용 |
|------|------|
| 목적 | 조직 단위별 고객경험 성과 측정 및 개선 관리 |
| 측정 지표 | NPS (Net Promoter Score) |
| 분석 관점 | 조직 단위별, 고객경험단계별 |

### 4.2 조직 단위별 NPS 측정

자가진단은 다음 **4개 조직 단위**로 NPS를 측정한다.

| 조직 단위 | 설명 | 분석 활용 |
|----------|------|----------|
| 영업추진그룹 | 최상위 지역 조직 단위 | 전사 차원 지역별 성과 비교 |
| PG/RG | 지역영업그룹 하위 조직 | 지역 내 본부별 성과 비교 |
| 영업점 | 개별 영업점 단위 | 영업점별 세부 성과 관리 |
| 직원 | 개별 직원 단위 | 직원별 고객응대 성과 관리 |

### 4.3 고객경험단계별 NPS 분석

각 조직 단위에서 **고객경험단계별로 NPS를 세분화**하여 분석할 수 있다.

| 고객경험단계 | 측정 내용 |
|-------------|----------|
| 내점 방문 | 영업점 접근성, 외부 환경 |
| 대기 | 대기 시간, 대기 공간 환경 |
| 상담 | 직원 전문성, 친절도 |
| 업무 처리 | 처리 속도, 정확성 |

### 4.4 자가진단 활용

```
[조직 선택] → [조직 단위별 NPS 확인] → [고객경험단계별 NPS 확인] → [문제 영역 파악] → [개선 활동 수행]
```

**활용 예시**:
* **영업추진그룹 비교**: 전사 차원에서 지역별 고객경험 성과 순위 파악
* **PG/RG 내 영업점 비교**: 동일 본부 내 영업점 간 성과 격차 분석
* **영업점 내 경험단계 분석**: 특정 영업점의 취약 경험단계 파악
* **직원별 성과 관리**: 우수 직원 사례 공유, 개선 필요 직원 코칭

### 6.3 고객경험요소
**정의**: 고객경험요소는 VoC(Voice of Customer) 텍스트에서 추출한 서비스품질요소를 유지하고 향상시키기 위한 구체적인 조치나 활동요소이다.
서비스품질요소보다 더 세부적인 단위로, 고객이 실제로 언급한 고유명사 기반의 개선 대상을 식별한다.

**구조적 위치**:
```
채널 → 고객경험단계 → 서비스품질요소 → 고객경험요소
(예) 플랫폼 → 로그인 → 인증 다양성 → 모바일OTP, 국민인증서, 간편비밀번호
(예) 영업점 → 대기 → 대기시간 → 스마트대기, 사전예약
(예) 고객센터 → 응대 → 전문성 → ARS메뉴, 콜백서비스
```

**추출 방식**:
* VoC 텍스트를 LLM 기반으로 분류할 때 함께 추출
* 고객이 직접 언급한 서비스명·기능명·상품명을 고유명사 기반으로 식별
* 예: "모바일OTP 인증이 자꾸 실패해요" → 고객경험요소: **모바일OTP**

**활용**:
* **개선과제 구체화**: 서비스품질요소(인증 다양성) 수준에서는 "인증 개선 필요"라고만 할 수 있지만, 고객경험요소(모바일OTP) 수준에서는 "모바일OTP 인증 성공률 개선"이라는 구체적 과제 도출 가능
* **빈도 분석**: 특정 고객경험요소가 얼마나 자주 언급되는지, 불만/칭찬 비율은 어떤지 분석
* **트렌드 파악**: 새롭게 등장하는 고객경험요소를 통해 신규 이슈를 조기에 발견

**고객경험요소 예시**:

| 채널 | 서비스품질요소 | 고객경험요소 예시 |
|------|-------------|--------------|
| 플랫폼 | 인증 다양성 | 모바일OTP, 국민인증서, 간편비밀번호, 생체인증 |
| 플랫폼 | 이체 신속성 | 간편송금, 자동이체, 해외송금 |
| 영업점 | 대기시간 | 스마트대기, 사전예약, 번호표시스템 |
| 고객센터 | 연결 용이성 | ARS메뉴, 콜백서비스, 챗봇상담 |
| 상품 | 상품 이해도 | 청년적금, 주택담보대출, 신용카드혜택 |

## 7. 설문 문항 구조
모든 설문은 다음 **단계형 구조**를 따른다.

### 7.1 1단계: 채널 추천 여부(NPS)
* “해당 채널(은행/앱/영업점/고객센터/상품)을 추천하시겠습니까?”
* 0~10점 기준 NPS 점수 측정

---

### 7.2 2단계: 추천/비추천 이유 (영향 경험 단계 도출)

* 추천 또는 비추천에 **영향을 준 고객경험단계 선택**
* 해당 이유에 대한 **자유 텍스트 의견 작성**

---

### 7.3 3단계: 고객경험단계별 추천 점수

* 선택한 고객경험단계 각각에 대해:

  * 추천 점수(NPS 방식)
  * 영향 서비스품질요소 선택
  * 구체적인 경험 텍스트 작성

---

### 7.4 4단계: 텍스트 감성 및 유형 분류

모든 텍스트 응답은 다음 기준으로 분류한다.

| 구분 | 유형           |
| -- | ------------ |
| VOC유형 | 칭찬 / 불만 / 개선 / 기타 |
| 감정 | 부정 / 긍정      |

## 8. 주요 측정 지표 정의 및 계산 공식
### 8.1 NPS (Net Promoter Score)
**정의**: 고객 충성도 및 추천 의향을 나타내는 대표 지표
**계산 공식**:
```
NPS = (추천 고객 수 / 전체 응답 수) × 100 - (비추천 고객 수 / 전체 응답 수) × 100
``
**고객 분류 기준**:
| 점수 구간 | 분류 | 설명 |
|----------|------|------|
| 9~10점 | 추천 고객 (Promoter) | 적극적으로 추천할 의향이 있는 고객 |
| 7~8점 | 중립 고객 (Passive) | 만족하지만 적극적 추천 의향은 낮음 |
| 0~6점 | 비추천 고객 (Detractor) | 불만족하여 추천하지 않을 고객 |

**계산 예시**:
- 전체 응답: 100명
- 추천 고객(9~10점): 40명 → 40%
- 중립 고객(7~8점): 30명 → 계산에서 제외
- 비추천 고객(0~6점): 30명 → 30%
- **NPS = 40% - 30% = +10**

**NPS 해석 원칙**:
NPS는 절대 수치가 아닌 **상대적 맥락**으로 해석한다.
* TD: 타행 대비 Gap(+/- 부호), 업계 내 순위(분모 명시), 벤치마크 은행 대비 차이로 해석
* BU: 전월 대비 증감(%p), 월간 추이 변화, 목표치 대비 달성도로 해석
---
### 8.2 NSS (Net Satisfaction Score)
**정의**: 특정 서비스나 경험 단계의 고객의 감정수준 측정 지표
**계산 공식**:
NSS = (긍정 VoC 비율) - (부정 VoC 비율)
```
---
### 8.3 CCI (Customer Complaint Index)
**정의**: 분류된 VoC 중 불만 응답의 비율 지표
**계산 공식**:
```
CCI = (불만으로 분류된 VoC 수 / 전체 VoC 수) × 100
```
**활용**: 불만 비율 파악, 시계열 추이 분석
---
### 8.4 영향도 (Impact Index)
**정의**: 채널 NPS를 고객경험단계별로 **분해**한 지표. 각 고객경험단계가 채널 NPS에 얼마나 기여(또는 저하)하는지를 보여준다.
**핵심 개념**:
> 영향도의 **분모**는 해당 경험단계를 선택한 고객 수가 아니라, **상위 개념(채널 또는 고객경험단계)의 전체 응답자 수**이다.
>
> 따라서 개별 영향도를 **단순 합산**하면 상위 NPS가 된다. (가중합이 아님)

영향도 산출 공식

| 구분 | 분해 대상 | 영향요인 단위 | 영향도 공식 |
|----------|----------|-------------|-----------|
| 채널 영향도 분석 | 채널 NPS | 고객경험단계 | (해당 경험단계 선택 추천자 - 비추천자) / **전체 채널 응답자 수** × 100 |
| 경험단계 영향도 분석 | 경험단계 NPS | 서비스품질요소 | (해당 품질요소 선택 추천자 - 비추천자) / **전체 경험단계 응답자 수** × 100 |

```
상위 NPS = Σ 영향도_i   (단순 합산)
```

** 산출 원리**:

영향도의 분모는 항상 **상위 개념의 전체 응답자 수**이다. 채널 영향도에서는 "해당 경험단계를 선택한 추천자/비추천자"를 세되, 분모는 채널 전체 응답자를 사용한다. 경험단계 영향도에서는 "해당 서비스품질요소를 선택한 추천자/비추천자"를 세되, 분모는 경험단계 전체 응답자를 사용한다. 이렇게 분모를 통일하므로, 개별 영향도를 모두 더하면 상위 NPS가 된다.

**계산 예시 (영업점 채널)**:

전체 응답: 200명, 영업점 채널 NPS: +15

1. 고객들이 추천/비추천 이유로 선택한 경험단계:
   - "대기" 선택: 50명
   - "상담" 선택: 80명
   - "업무처리" 선택: 70명

2. 각 경험단계별 영향도 계산 (**분모 = 전체 200명**):
   - **대기 영향도**: (추천자 10명 - 비추천자 25명) / **200** × 100 = **-7.5**
   - **상담 영향도**: (추천자 50명 - 비추천자 10명) / **200** × 100 = **+20.0**
   - **업무처리 영향도**: (추천자 30명 - 비추천자 25명) / **200** × 100 = **+2.5**

3. 분해 검증 (단순 합산):
   - 채널 NPS = (-7.5) + (+20.0) + (+2.5) = **+15.0** ✓
   - 개별 영향도의 합 = 채널 NPS (정확히 일치)

4. 해석:
   - 대기(-7.5): NPS를 끌어내리는 주요 원인 → **즉시 개선 필요**
   - 상담(+20.0): NPS를 가장 크게 끌어올리는 강점 → 유지·강화
   - 업무처리(+2.5): 약한 긍정 기여 → 모니터링

**영향도 활용**:
* 영향도가 음수인 요인: 채널 NPS를 끌어내리는 원인 → **우선 개선 대상**
* 영향도가 양수인 요인: 채널 NPS를 끌어올리는 강점 → 유지·강화
* 영향도 절대값이 큰 경험단계가 채널 NPS에 미치는 영향이 크다
* 같은 원리로 경험단계 NPS → 서비스품질요소 영향도로 2차 분해 가능

## 10. 개선과제 도출 체계
### 10.1 IPA 분석 (중요도–성과 GAP 매트릭스) — TD 전용
> IPA 분석은 **TD 조사 전용**이다. BU 조사에는 타행 벤치마크 데이터가 없으므로 IPA 분석이 불가하다.
> BU에서 유사한 분석이 필요한 경우, 영향도(음수 요인) + VoC 불만 분석으로 대체한다.

분석 기준:
* X축: NPS 영향도 GAP (KB 영향도 - 벤치마크 영향도) — 성과 축
* Y축: NPS 중요도 — 중요도 축

※ 여기서 "NPS 중요도"는 8.4절의 "영향도(Impact Index)"와 다른 개념이다. 영향도는 NPS를 분해한 각 요인의 기여도이고, NPS 중요도는 IPA 분석에서 해당 요인이 전체 NPS에 미치는 중요도를 별도 공식으로 산출한 지표이다 (10.3절 참조).

| 영역      | 의미        | 전략     |
| ------- | --------- | ------ |
| 고중요도·저성과 | 중점개선 | 즉시 개선  |
| 고중요도·고성과 | 유지관리     | 유지·확대  |
| 저중요도·저성과 | 점진개선       | 선택적 개선 |
| 저중요도·고성과 | 현상유지     | 유지     |

---

### 10.2 벤치마크 방식

**벤치마크 은행 산정 기준**:
* KB국민은행 기준으로 **채널별 NPS 점수 직상위 1개 기관**을 벤치마크로 선정
* 채널마다 벤치마크 은행이 다를 수 있음

**산정 예시**:
| 채널 | KB국민은행 NPS | 타행 NPS 순위 | 벤치마크 은행 |
|------|---------------|--------------|--------------|
| 플랫폼 | 35 | A은행(42), B은행(38), KB(35), C은행(30) | B은행 (직상위) |
| 영업점 | 28 | B은행(40), A은행(32), C은행(29), KB(28) | C은행 (직상위) |
| 고객센터 | 45 | KB(45), A은행(42), B은행(38) | A은행 (KB가 1위일 경우 2위사) |

**활용**:
* 벤치마크 은행과의 **갭(GAP) 점수** = 벤치마크 NPS - KB국민은행 NPS
* GAP이 클수록 개선 필요도가 높음
* IPA 분석의 X축(BM 대비 성과)으로 활용

### 10.3 NPS 중요도

**정의**: 특정 영향요인이 각 채널 NPS에 미치는 중요도를 나타내는 지표

**계산 공식**:
```
NPS 중요도 = (영향요인 응답률 + 영향요인 비추천응답률) / 2
```

* **영향요인 응답률**: 해당 영향요인을 선택한 고객 수 / 전체 응답 고객 수
* **영향요인 비추천응답률**: 해당 영향요인을 선택한 비추천 고객 수 / 전체 비추천 고객 수

**계산 예시**:
- 전체 응답: 100명, 비추천 고객: 30명
- "대기" 선택 고객: 50명 (응답률 50%)
- "대기" 선택 비추천 고객: 20명 (비추천응답률 66.7%)
- **대기 NPS 중요도 = (50% + 66.7%) / 2 = 58.3%**

**활용**:
* 중요도가 높은 요인 = 전체 NPS에 큰 영향을 미치는 요인
* IPA 분석의 Y축으로 활용하여 개선 우선순위 결정

### 10.4 IPA 4분면 기준

4분면 구분은 **평균값**을 기준으로 함:

```
                    NPS 중요도 (Y축)
                         ↑
         [중점개선]      │      [유지관리]
    고중요도·저성과      │     고중요도·고성과
         (즉시 개선)     │      (유지·확대)
    ─────────────────────┼─────────────────────→ NPS 영향도 GAP (X축)
         [점진개선]      │      [현상유지]                       (KB-BM)
    저중요도·저성과      │     저중요도·고성과
        (선택적 개선)    │       (유지)
```

* **X축 기준선**: 전체 영향요인 NPS 영향도 GAP의 평균 (좌측=BM 대비 열위, 우측=BM 대비 우위)
* **Y축 기준선**: 전체 영향요인 NPS 중요도의 평균 (상단=고중요도, 하단=저중요도)

## 11. 고객경험 관리활동 (BU 전용)

### 11.1 고객경험 관리활동 개요

고객경험 관리활동은 **BU 조사에서 수집된 VoC를 기반으로 관련 부서에 개선과제를 배분하고, 검토·처리 현황을 관리**하는 체계이다. TD 조사는 배분 대상이 아니므로 개선조치검토 체계가 적용되지 않는다.

### 11.2 고객경험관리 프로세스
```
VoC 수집 → 고객경험요소 분류 → 고객경험단계 기반 관리부서 배분 → 부서 검토 → 고객피드백 발송 → 개선 실행
```
### 11.3 배분 및 검토 상태
| 구분 | 상태값 | 설명 |
|------|--------|------|
| 과제진행상태 | 검토완료 | 배분된 과제에 대해 부서가 검토를 완료한 상태 |
| | 미처리 | 배분되었으나 아직 검토하지 않은 상태 |
| | 검토기한만료 | 검토 기한이 경과한 상태 |
| 검토구분 (검토완료 시) | 현행유지 | 현재 운영 방식을 유지하기로 결정 |
| | 개선예정 | 개선이 필요하여 실행 계획을 수립 |
| | 개선불가 | 구조적·정책적 사유로 개선이 어려운 건 |

### 11.4 주요 관리 지표

* **검토율** = 검토완료 건수 / 배분 건수 × 100
고객 피드백 발송률 = 피드백 발송건수 / 검토완료 건수 x 100

### 11.5 분석 관점

* **부서별 현황**: 개선사업그룹·개선부서 단위로 배분건수, 처리율, 검토구분별(현행유지, 개선예정, 개선불가) 과제 건수, 피드백 발송률 등 파악
* **서비스품질요소별 현황**: 채널·고객경험단계·서비스품질요소 단위로 배분·검토 현황 분석

|----------|----------|------------|
| "가장 시급하게 개선해야 할 영역은?" | IPA 분석 수행 → 고영향·저성과 영역 확인 → 해당 영역의 VoC 분석 | NPS, 영향도, IPA 매트릭스 |
| "경쟁사 대비 개선이 필요한 부분은?" | TD 조사 기준 GAP 분석 → 열위 채널/경험단계 확인 → BU 조사로 세부 원인 파악 | TD/BU 조사 결과, 벤치마크 |

### 12.5 분석 프로세스 가이드

**기본 분석 흐름**:
```
1. 전체 현황 파악: 채널별 NPS 확인
2. 문제 영역 특정: 낮은 NPS 채널의 경험단계별 영향도 분석
3. 세부 원인 파악: 해당 경험단계의 서비스품질요소별 영향도 분석
4. 실제 고객 목소리 확인: 관련 VoC 텍스트 분석
5. 개선방향 도출: IPA 분석 기반 우선순위화
```

**비교 분석 시 고려사항**:
* 시계열 비교: 동일 기간(전월, 전분기, 전년) 대비
* 세그먼트 비교: 동일 조건의 다른 그룹과 비교
* 벤치마크 비교: 타행 평균 또는 업계 평균과 비교

## 13. VoC 텍스트 분류 체계

### 13.1 분류 방식

VoC 텍스트는 **자체 분류 모델** 또는 **LLM 기반 분류**를 통해 자동 분류된다.

### 13.2 분류 결과 항목

| 분류 차원 | 분류 값 | 설명 |
|----------|--------|------|
| VOC 유형 | 불만 | 서비스/경험에 대한 부정적 피드백 |
| | 칭찬 | 서비스/경험에 대한 긍정적 피드백 |
| | 개선 | 구체적인 개선 제안 또는 요청 |
| | 기타 | 위 유형에 해당하지 않는 의견 |
| 감정 | 긍정 | 긍정적 감정이 담긴 텍스트 |
| | 부정 | 부정적 감정이 담긴 텍스트 |
| | 중립 | 중립적 감정이 담긴 텍스트 |

### 13.3 분류 결과 활용

* **CCI 산출**: 불만 유형 비율 집계
* **경험단계/서비스품질요소 매핑**: 분류된 VoC를 구조화된 체계에 연결
* **트렌드 분석**: 유형별 추이 모니터링
* **대표 VoC 추출**: 분석 결과 설명을 위한 실제 고객 목소리 제시
